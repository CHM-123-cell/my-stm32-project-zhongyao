# 中药自动分拣系统 (STM32F103C8T6)

基于 STM32F103C8T6 的袋装中药**双闸门垂直验证式精确计数 / 分拣下料**系统。

通过「上闸放料 + 上光电计数」与「下闸复核 + 下光电复验」两段式流程，对袋装药品做
精确计数与批次校验，计数达标或超时后自动闸门闭合、批次完成。运行参数可通过串口在线整定。

---

## 1. 硬件清单

| 部件 | 型号 / 规格 | 说明 |
|------|-------------|------|
| MCU | STM32F103C8T6 | Cortex-M3，72MHz（本工程锁定 8MHz HSI，见第 7 节） |
| 上闸舵机 | MG996R | 上闸门驱动 |
| 下闸舵机 | MG996R | 下闸门驱动 |
| 上光电传感器 | E3F-5C1 ×1 | 上料口计数（经 PC817 光耦隔离） |
| 下光电传感器 | E3F-5C1 ×1 | 落料口复核（经 PC817 光耦隔离） |
| 显示 | OLED 0.96" SSD1306 | I2C 接口，软件模拟时序 |
| 通信 | USART1 @ 9600 8N1 | 串口指令 / 状态上报 |

> ⚠️ **舵机供电**：MG996R 工作电流峰值可达 ~1A/个，必须**独立 5V / ≥2A 电源**驱动，
> 严禁从 MCU 的 3.3V 或板载 LDO 取电，否则会拉垮 MCU 电源导致复位 / 舵机无力。

## 2. 引脚分配

| MCU 引脚 | 复用 / 外设 | 方向 | 连接对象 | 说明 |
|----------|-------------|------|----------|------|
| PA0 | TIM2_CH1 (PWM) | 输出 | 上闸舵机信号线 | 50Hz，0.5~2.5ms 对应 0°~180° |
| PA1 | TIM2_CH2 (PWM) | 输出 | 下闸舵机信号线 | 同上 |
| PB0 | EXTI0 (GPIO 输入，IPU) | 输入 | 上光电 (PC817 输出) | 下降沿触发；遮挡=拉低 |
| PB1 | EXTI1 (GPIO 输入，IPU) | 输入 | 下光电 (PC817 输出) | 下降沿触发；遮挡=拉低 |
| PA9 | USART1_TX | 输出 | USB-TTL 的 RX | 9600 8N1 |
| PA10 | USART1_RX | 输入 | USB-TTL 的 TX | 9600 8N1 |
| PB8 | GPIO (软件 I2C) | 输出 OD | OLED SCL | 从机地址 0x3C |
| PB9 | GPIO (软件 I2C) | 输出 OD | OLED SDA | 从机地址 0x3C |

> 注：E3F-5C1 为 NPN 常开型，遮挡时输出导通拉低；PC817 二次隔离后接入 MCU 上拉输入，
> 故中断采用 `EXTI_Trigger_Falling`（遮挡→下降沿）。若实测计数翻倍或反向，将 `sensor.c`
> 中 `EXTI_Trigger_Falling` 改为 `EXTI_Trigger_Rising` 即可（取决于光耦输出端极性接法）。

## 4. 工作原理 — 双闸门状态机

```
                ┌─────────── @S 启动 (需先 @数字 设目标) ───────────┐
                ▼                                                    │
   [IDLE] ───────────────────────────────────────────────────► [FEED]
   待机                                                          上闸开(90°)
   设目标/调参                                                   上光电计数
                                                                  │
                                             计数≥目标 ──┐       │
                                            或 超时(TO) ──┴──────┤
                                                                  ▼
                                                               [DROP]
                                                          上闸关(180°)
                                                          等 D ms 下落补偿
                                                                  │
                                                                  ▼
                                                              [VERIFY]
                                                          下闸开(90°)
                                                          下光电复核
                                             复核≥上计数 ─┐            │
                                             或 超时 ─────┴────────────┤
                                                                  ▼
                                                              [BLINK]
                                                          闪烁提示 ~2.4s
                                                                  │
                                                                  ▼
                                                              [DONE]
                                                      上报完成(UP/LO/TGT)
                                                      自动复位回 [IDLE]
```

- **角度约定**：`90° = 闸门开`，`180° = 闸门闭`（见 `servo.h` 注释，与 `main.c` 一致）。
- **校验逻辑**：下光电 `Sensor_Lo` 需 ≥ 上光电 `Sensor_Up`；若 `LO != UP` 标记 `bad=1`
  （MISMATCH），OLED 末行显示 `MISMATCH!`。
- **急停 `@R`**：舵机立即归 180°（双闸关闭），清零计数与全部运行状态后回到 IDLE。

## 5. 串口指令集 (USART1 @ 9600 8N1)

所有指令以 `@` 开头，回车 `\r` 结束。支持大小写。

| 指令 | 示例 | 功能 | 取值范围 / 备注 |
|------|------|------|-----------------|
| `@<数字>` | `@30` | 设置目标计数 | 1 ~ 65535；发送前先设定 |
| `@S` | `@S` | 启动批次 | 需 `目标>0` 且处于 IDLE，否则回 `TGT=0` / `BUSY` |
| `@R` | `@R` | 急停并复位 | 双闸关闭、清计数与状态 |
| `@D<ms>` | `@D2000` | 设置下落补偿延时 D | 100 ~ 10000 ms（默认 2000） |
| `@T<ms>` | `@T5000` | 设置超时 T | 500 ~ 60000 ms（默认 5000） |
| `@?` | `@?` | 查询状态 | 回 `TGT/DM/TO/UP/LO/ST` |

上报样例：

```
===== V9.5 =====
[S] TGT=30
[OK] UP=30
[V]
[OK] LO=30
=== DONE TGT:30 UP:30 LO:30 ===
```

## 6. 编译与烧录 (Keil MDK)

1. 用 Keil μVision 打开 `test.uvprojx`（器件选 **STM32F103C8**）。
2. **必须勾选 `Use MicroLIB`**：
   `Project → Options for Target → Target → 勾选 "Use MicroLIB"`。
   > `Serial.c` 使用 `printf`/`vsprintf` + `fputc` 重定向，未勾 MicroLIB 会导致
   > 串口上报链接失败或引入 semihosting 卡死。工程已默认开启该选项。
3. 编译（`F7`），烧录（`F8`，需 ST-Link / CMSIS-DAP）。
4. 打开串口助手（9600 8N1），发送指令即可。

> 工程未使用 HSE 外部晶振：`main.c` 内 `RCC_DeInit()` 后锁定 **HSI 8MHz**，
> `SysTick_Config(8000)` 产生 1ms 节拍（`g_sys_tick`）。无外部晶振也能稳定启动，
> 但**全系统运行在 8MHz**（非标称 72MHz）。如需升频，需改 HSE 配置并相应调整
> `SysTick_Config` 与 `TIM_Prescaler`。

## 7. 上板前必查事项

| # | 项目 | 说明 |
|---|------|------|
| 1 | 舵机电源 | 独立 5V/≥2A，不可与 MCU 3.3V 共用 |
| 2 | 舵机脉宽标定 | 用示波器实测 90°/180° 是否对应 1.5ms/2.5ms；不同舵机有偏差，必要时调 `servo.c` 的 `500+deg*2000/180` |
| 3 | 传感器极性 | 遮挡计数翻倍/反向 → 改 `sensor.c` 的 EXTI 触发沿 |
| 4 | 消抖窗口 | `SENSOR_DEBOUNCE_MS=200` 防抖；药袋间隔 < 200ms 会漏计，按真实下落节拍调小 |
| 5 | 光耦隔离 | E3F-5C1 → PC817 → MCU，确认输出端接 PB0/PB1 且上拉到 3.3V |

## 8. 已知限制 / 后续可优化

- 系统锁定 8MHz（见第 6 节），如需更高舵机刷新或更快串口吞吐需升频。
- 计数与超时均依赖 `g_sys_tick`（SysTick 1ms）。SysTick 中断被长时间屏蔽时会漂移。
- 双舵机 PWM 由 TIM2 同一计数器生成，相位一致；如需错相可改独立定时器。
- 无看门狗（IWDG/WWDG）保护，建议量产固件加入独立看门狗防死机。
- 串口接收缓冲已加 `pRxPacket < 99` 越界保护，但协议无校验和，强干扰环境建议加 CRC。
